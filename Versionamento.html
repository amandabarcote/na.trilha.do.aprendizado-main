Software de Versionamento

Para que serve o Git?
O Git serve para organizar as versões de um software, organizar as versões, versionamento e melhora a organização da equipe, uma vez que o software se responsabiliza pela organização.


Controle de versões


Sobre os tipos de Versionamentos (VCS)
Antes de falar dos tipos, tenho que explicar o basico sobre como ocorre o vercionamento, ele é uma versão de seu código, ele pode ser guardado em sua propria maquina ou mandado para um servidor central/remoto, o Git oferece o vercionamento tanto local quando para um servidor remoto. Na imagem acima vemos ilustrado a conexão entre um repositorio local com um repositorio remoto (o repositorio falando em termos brutos é uma pasta com seus arquivos).

O vercionamento centralizado/linar é uma maquina fazer o versionamento num servidor central, é como se todos os repositorios de um escritorio fossem ligados um servidor proprio do escritorio, para ter sucesso é necessario estar sempre conectado ao servidor central, esse sistema é mais antigo porem ainda está em uso, os VCS que representão esse sistema é o CVS e SVN. Ele não tem versionamento local e é totalmente dependente do servidor.

O versionamento distribuido tem um repositorio local e um repositorio remoto, o usuario tem o controle se quer atualizar o repositoro remoto ou se quer deixar o repositorio apenas em sua maquina, este sistema é mais novo e melhor, os softwares mais notaveis são o BitKeep e o Git.

Git x GitHub


Qual a diferença entre Git e GitHub?
Em poucas palavras a diferença é que, o Git faz o versionamento e o GitHub hospeda esse versionameto, os dois são distintos mas se completam, junto os dois se fundem e formam o melhor software de versionamento (na minha opinião), serve para versionar qualquer codigo fonte, com o GitHub é possivel acessar seu codigo em qualquer lugar, o GitHub pode ser considerado uma rede social tambem.
